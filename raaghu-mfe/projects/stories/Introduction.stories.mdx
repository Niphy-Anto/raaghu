### Welcome to raaghu design system
<div className="content">
 <strong>raaghu</strong> is an open-source Angular based micro front-end design system. It is an online repository of elements,
  components, pages and charts to developers. raaghu employs Bootstrap5 and Storybook features to develop highly customizable UI 
  components and built-in pages. 

<img className="demo" src={demo} alt="flow" />

Storybook provides a first-hand visual reference of elements and components built by raaghu. Snippets for the elements
 allow you to test and design your UI faster and efficiently. 
</div>

### Why raaghu?

<div className="content"> raaghu design system bridges the gaps between design and development while maintaining a great degree of standardization.</div>
<ul className="content">
<li><strong>Seamless UI:</strong>  raaghu contains ready to use templates compatible with ASP.Net Zero and ABP commercial framework. Fully customizable with
 the help of our Adobe XD and figma (coming soon!) kits.</li>
<li><strong>Charts, Layouts and Grids:</strong>With more than 300+ angular based elements, components , pages and charts , raaghu offers a wide array to meet 
all your requirements for building customisable applications.</li>
</ul> 

### How raaghu Works?
<div className="content">
raaghu leverages micro-front-end capabilities by breaking the page design into the components. It applies the desired desired CSS or default properties to the 
elements, components and pages available in the raaghu.
<img className="how" src={image} alt="flow" />
Thought the Ci/Cd pipelines the components are deployed to the wireframe design and form a page
</div>

### Features 
<ul className="content">
<li><strong>Fast: </strong>Achieve up to 3X acceleration with front end development through modular creation and component composition.</li>
<li><strong>Consistent : </strong>Get a unified experience via smooth integration, code maintenance and consistency in UI.</li>
<li><strong>Scalable : </strong>Provides better scalability as each codebase is much simpler to build, test and maintain due to clarity of work allocation.</li>
<li><strong>Maintainable : </strong> Replace, rewrite and update components even for maintaining large applications. Components allow for easier user testing.</li>
<li><strong>Flexible : </strong>Components are inherently independent of the tech stack allowing multiple components and features to be deployed rapidly.</li>
<li><strong>Pixel-perfect : </strong> Designed with pixel perfect precision and accessibility in mind, with multiple well designed and scalable User Interfaces.</li>    
</ul>
